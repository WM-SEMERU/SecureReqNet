'### Summary
  
Merge request reports: "Something went wrong on our end" if the project has permissions for Merge Requests set to "Only Project Members" for internal projects.

### Steps to reproduce

* Create a new project (or use an existing project)
* Set the projects visibility to "internal"
* Set project's permission "Merge requests" to "Only Project Members"
* Ask a friend (or any other non-project-member) with at an account to come help you out
* Ask friend to: Login
* Ask friend to: Create a fork of the project
* Ask friend to: "Create a new merge request" (e.g., from the projects homepage) to the original one

### Example Project

Here is an example project:

https://gitlab.com/zencore/testing-merge-requests

Try to:

* Log into GitLab.com
* Fork the project
* Try to create merge request to the original project

### What is the current *bug* behavior?

* The error "Something went wrong on our end" appears after trying to create a merge request

### What is the expected *correct* behavior?

* Some text explaining that merge requests are not available for this project
* **OR**
* "Create Merge Mequest" buttons should not be shown when users don't haver permission to do so (as is the case for non-logged in users)
